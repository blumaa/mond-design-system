# Design Tokens

Design tokens are the visual design atoms of the Mond Design System. They store visual design attributes like colors, spacing, typography, and more. These tokens ensure consistency across all components and make theming straightforward.

## üé® Colors

### Semantic Colors
Semantic colors automatically adapt to light and dark themes, providing consistent meaning across your application.

#### Text Colors
```tsx
color="textPrimary"    // Main content text
color="textSecondary"  // Supporting information
color="textTertiary"   // Less important details
color="textDisabled"   // Inactive content
color="textInverse"    // Text on dark backgrounds
color="textLink"       // Interactive links
color="textSuccess"    // Positive feedback
color="textWarning"    // Cautionary information
color="textError"      // Problem notifications
```

#### Surface Colors
```tsx
backgroundColor="surfaceBackground"  // Main background
backgroundColor="surfaceElevated"    // Elevated surfaces
backgroundColor="surfaceOverlay"     // Modal overlays
backgroundColor="surfaceCard"        // Card backgrounds
backgroundColor="surfaceInput"       // Form input backgrounds
backgroundColor="surfaceDisabled"    // Disabled element backgrounds
```

#### Interactive Colors
```tsx
// Primary Actions
backgroundColor="interactivePrimaryBackground"        // Button background
backgroundColor="interactivePrimaryBackgroundHover"   // Button hover state
backgroundColor="interactivePrimaryBackgroundPressed" // Button pressed state
color="interactivePrimaryText"                        // Button text

// Secondary Actions
backgroundColor="interactiveSecondaryBackground"      // Secondary button
borderColor="interactiveSecondaryBorder"              // Secondary button border
color="interactiveSecondaryText"                      // Secondary button text
```

### Direct Color Tokens
Use specific color values when semantic colors don't apply:

```tsx
// Blue palette
color="blue50"    // Lightest blue
color="blue500"   // Medium blue
color="blue900"   // Darkest blue

// Gray palette
color="gray100"   // Light gray
color="gray500"   // Medium gray
color="gray800"   // Dark gray

// Status colors
color="red500"    // Error red
color="green500"  // Success green
color="amber500"  // Warning amber
```

## üìè Spacing

The spacing scale provides consistent spacing throughout your application:

```tsx
// Spacing tokens (based on 0.25rem = 4px)
spacing="0"   // 0px
spacing="1"   // 4px (0.25rem)
spacing="2"   // 8px (0.5rem)  
spacing="3"   // 12px (0.75rem)
spacing="4"   // 16px (1rem) - base spacing unit
spacing="5"   // 20px (1.25rem)
spacing="6"   // 24px (1.5rem)
spacing="8"   // 32px (2rem)
spacing="10"  // 40px (2.5rem)
spacing="12"  // 48px (3rem)
spacing="16"  // 64px (4rem)
spacing="20"  // 80px (5rem)
spacing="24"  // 96px (6rem)
```

### Usage Examples
```tsx
<Box
  padding="4"                                  // padding: 1rem
  margin="2"                                   // margin: 0.5rem
  paddingLeft="6" paddingRight="6"            // padding-left & padding-right: 1.5rem
  marginTop="8" marginBottom="8"              // margin-top & margin-bottom: 2rem
  paddingTop="3"                              // padding-top: 0.75rem
  marginBottom="5"                            // margin-bottom: 1.25rem
  gap="4"                                     // gap: 1rem (for flexbox/grid)
>
  Content
</Box>
```

## üåô Theme Integration

### Using ThemeProvider
Wrap your application with `ThemeProvider` to enable theme switching:

```tsx
import { ThemeProvider } from '@mond-design-system/theme';
import { mondTheme } from '@mond-design-system/theme';

function App() {
  const [colorScheme, setColorScheme] = useState<'light' | 'dark'>('light');

  return (
    <ThemeProvider brandTheme={mondTheme} colorScheme={colorScheme}>
      <Box
        backgroundColor="surfaceBackground"
        color="textPrimary"
        borderColor="borderDefault"
      >
        Themed content automatically adapts
      </Box>
    </ThemeProvider>
  );
}
```

### Using Semantic Tokens
Use semantic tokens that automatically adapt to the current theme:

```tsx
// ‚úÖ Recommended - adapts to theme via ThemeProvider
<Box
  backgroundColor="surfaceBackground"
  color="textPrimary"
  borderColor="borderDefault"
>
  Themed content
</Box>

// ‚ùå Not recommended - static colors
<Box backgroundColor="white" color="black">
  Static content
</Box>
```


## üéØ Best Practices

### 1. Prefer Semantic Tokens
```tsx
// ‚úÖ Good - semantic meaning
color="textPrimary"
backgroundColor="surfaceBackground"

// ‚ùå Avoid - hardcoded values
color="gray900"
backgroundColor="white"
```

### 2. Use Consistent Spacing
```tsx
// ‚úÖ Good - uses spacing scale
<Stack spacing="4">
  <Box marginBottom="2">Item 1</Box>
  <Box marginBottom="2">Item 2</Box>
</Stack>

// ‚ùå Avoid - arbitrary values
<div style={{ marginBottom: '13px' }}>
  Item 1
</div>
```

### 3. Follow Typography Scale
```tsx
// ‚úÖ Good - uses type scale
<Heading size="2xl" weight="bold">Title</Heading>
<Text fontSize="lg">Large body text</Text>

// ‚ùå Avoid - custom sizes
<h1 style={{ fontSize: '28px' }}>Title</h1>
```

### 4. Theme Consistency
```tsx
// ‚úÖ Good - wrapped in ThemeProvider
<ThemeProvider brandTheme={mondTheme} colorScheme={colorScheme}>
  <MyComponent />
</ThemeProvider>

// ‚ùå Inconsistent - missing ThemeProvider
<MyComponent />
```

Design tokens ensure your application maintains visual consistency and makes theming effortless. Always refer to these tokens when building custom components to maintain design system coherence.
