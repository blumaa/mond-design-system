# Design Tokens

Design tokens are the visual design atoms of the Mond Design System. They store visual design attributes like colors, spacing, typography, and more. These tokens ensure consistency across all components and make theming straightforward.

## üé® Colors

### Semantic Colors
Semantic colors automatically adapt to light and dark themes, providing consistent meaning across your application.

#### Text Colors
```tsx
color="text.primary"    // Main content text
color="text.secondary"  // Supporting information  
color="text.tertiary"   // Less important details
color="text.disabled"   // Inactive content
color="text.inverse"    // Text on dark backgrounds
color="text.link"       // Interactive links
color="text.success"    // Positive feedback
color="text.warning"    // Cautionary information
color="text.error"      // Problem notifications
```

#### Surface Colors
```tsx
color="surface.background"  // Main background
color="surface.elevated"    // Elevated surfaces
color="surface.overlay"     // Modal overlays
color="surface.card"        // Card backgrounds
color="surface.input"       // Form input backgrounds
color="surface.disabled"    // Disabled element backgrounds
```

#### Interactive Colors
```tsx
// Primary Actions
color="interactive.primary.background"        // Button background
color="interactive.primary.backgroundHover"   // Button hover state
color="interactive.primary.backgroundPressed" // Button pressed state
color="interactive.primary.text"             // Button text

// Secondary Actions  
color="interactive.secondary.background"      // Secondary button
color="interactive.secondary.border"          // Secondary button border
color="interactive.secondary.text"           // Secondary button text
```

### Direct Color Tokens
Use specific color values when semantic colors don't apply:

```tsx
// Blue palette
color="blue.50"    // Lightest blue
color="blue.500"   // Medium blue  
color="blue.900"   // Darkest blue

// Gray palette
color="gray.100"   // Light gray
color="gray.500"   // Medium gray
color="gray.800"   // Dark gray

// Status colors
color="red.500"    // Error red
color="green.500"  // Success green
color="amber.500"  // Warning amber
```

## üìè Spacing

The spacing scale provides consistent spacing throughout your application:

```tsx
// Spacing tokens (based on 0.25rem = 4px)
spacing="0"   // 0px
spacing="1"   // 4px (0.25rem)
spacing="2"   // 8px (0.5rem)  
spacing="3"   // 12px (0.75rem)
spacing="4"   // 16px (1rem) - base spacing unit
spacing="5"   // 20px (1.25rem)
spacing="6"   // 24px (1.5rem)
spacing="8"   // 32px (2rem)
spacing="10"  // 40px (2.5rem)
spacing="12"  // 48px (3rem)
spacing="16"  // 64px (4rem)
spacing="20"  // 80px (5rem)
spacing="24"  // 96px (6rem)
```

### Usage Examples
```tsx
<Box 
  p="4"        // padding: 1rem
  m="2"        // margin: 0.5rem  
  px="6"       // padding-left & padding-right: 1.5rem
  my="8"       // margin-top & margin-bottom: 2rem
  pt="3"       // padding-top: 0.75rem
  mb="5"       // margin-bottom: 1.25rem
  gap="4"      // gap: 1rem (for flexbox/grid)
>
  Content
</Box>
```

## üåô Theme Integration

### Using Semantic Tokens
The recommended approach is to use semantic tokens that automatically adapt:

```tsx
// ‚úÖ Recommended - adapts to theme
<Box 
  bg="surface.background"
  color="text.primary"
  borderColor="border.default"
  isDarkMode={isDarkMode}
>
  Themed content
</Box>

// ‚ùå Not recommended - static colors
<Box bg="white" color="black">
  Static content
</Box>
```

### Component Theme Props
Most components accept an `isDarkMode` prop for theme switching:

```tsx
<Text isDarkMode={isDarkMode} semantic="primary">
  This text adapts to the theme
</Text>

<Heading isDarkMode={isDarkMode} semantic="secondary">
  This heading also adapts
</Heading>
```

## üìñ Token Usage in Components

### Text Component
```tsx
<Text 
  variant="body-lg"        // Uses fontSize and lineHeight tokens
  weight="semibold"        // Uses fontWeight tokens
  semantic="primary"       // Uses semantic color tokens
  isDarkMode={isDarkMode}  // Enables theme adaptation
>
  Styled with design tokens
</Text>
```

### Box Component
```tsx
<Box
  p="4"                    // Uses spacing tokens
  bg="surface.card"        // Uses semantic color tokens  
  borderRadius="md"        // Uses radius tokens
  boxShadow="sm"           // Uses shadow tokens
  isDarkMode={isDarkMode}  // Enables theme adaptation
>
  Layout with design tokens
</Box>
```

## üéØ Best Practices

### 1. Prefer Semantic Tokens
```tsx
// ‚úÖ Good - semantic meaning
color="text.primary"
bg="surface.background"

// ‚ùå Avoid - hardcoded values
color="gray.900" 
bg="white"
```

### 2. Use Consistent Spacing
```tsx
// ‚úÖ Good - uses spacing scale
<Stack spacing="4">
  <Box mb="2">Item 1</Box>
  <Box mb="2">Item 2</Box>
</Stack>

// ‚ùå Avoid - arbitrary values
<div style={{ marginBottom: '13px' }}>
  Item 1
</div>
```

### 3. Follow Typography Scale
```tsx
// ‚úÖ Good - uses type scale
<Heading size="2xl" weight="bold">Title</Heading>
<Text fontSize="lg">Large body text</Text>

// ‚ùå Avoid - custom sizes
<h1 style={{ fontSize: '28px' }}>Title</h1>
```

### 4. Theme Consistency
```tsx
// ‚úÖ Good - passes theme prop
<MyComponent isDarkMode={isDarkMode} />

// ‚ùå Inconsistent - missing theme awareness
<MyComponent />
```

Design tokens ensure your application maintains visual consistency and makes theming effortless. Always refer to these tokens when building custom components to maintain design system coherence.
